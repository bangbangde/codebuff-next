# Repository Context

Before product, UI, or architecture work, read:

1. `docs/project.md`
2. `docs/current-milestone.md`
3. the active milestone README referenced by `docs/current-milestone.md`

For milestone creation, revision, closure, workflow changes, or questions about maintaining project context, also read `docs/ai-native-workflow.md`.

The current task or GitHub Issue defines the requested deliverable, the active Milestone Contract defines the phase boundary, and `docs/project.md` defines stable long-term direction. Use `docs/ai-native-workflow.md` for the complete source-ownership and conflict-resolution rules.

Milestone supporting files are non-binding and should be read only when the active milestone README or the current task references them.

## Change Boundaries

- Do not edit the Contract section of the active milestone unless the current task explicitly revises or reopens it.
- Update the active milestone's Tracking section when normal work changes milestone progress, open questions, or milestone-local decisions.
- Keep milestone scope, progress, and temporary decisions out of `docs/project.md`.
